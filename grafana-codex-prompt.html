Codex向け Grafana ダッシュボード作成プロンプト

Windows WSL2 と Docker Desktop 上で、Grafana REST API を使ってデモ用ダッシュボードを自動作成するためのプロンプト例です。

詳細版プロンプト

Codexに投げるプロンプト例です。このまま使えます。

Windows WSL2 上の Docker Desktop 環境で、Grafana の REST API を使ってデモ用ダッシュボードを自動作成してください。

要件:
- Docker Compose で Grafana を起動する
- ホスト側ポートは環境変数 GRAFANA_PORT で変更可能にする
- デフォルトは 3030 番を使う
- 管理者ユーザーは admin / admin
- Grafana HTTP API を使って Service Account を作成する
- Service Account Token を発行する
- Token を使って TestData datasource を作成または更新する
- `POST /api/dashboards/db` でダッシュボードを作成する
- 同じスクリプトを複数回実行しても壊れないようにする

作成するファイル:
- docker-compose.yml
- scripts/setup-grafana-wsl2.sh
- dashboards/ship-sensor-dashboard.json
- README.md

ダッシュボード要件:
- UID: ship-sensor-demo
- Title: Ship Sensor Dashboard - Demo
- refresh: 5s
- time range: now-6h to now
- datasource: grafana-testdata-datasource
- datasource uid: testdata

パネル:
1. Engine Temperature
   - type: timeseries
   - TestData random_walk
   - unit: celsius
   - thresholds: 60 yellow, 80 red

2. Fuel Pressure
   - type: timeseries
   - TestData random_walk
   - 2 series: Main Line, Sub Line
   - unit: pressurebar

3. Current Status
   - type: gauge
   - 表示値: RPM
   - unit: rpm
   - min: 0
   - max: 7000
   - thresholds: 4500 yellow, 6000 red

4. Atmospheric Pressure
   - type: stat
   - unit: pressurehpa
   - thresholds: 1015 yellow, 1030 red

5. Sensor Alert History
   - type: table
   - TestData csv_content
   - columns: time, sensor, severity, message
   - severity は INFO/WARN/CRITICAL を色分けする

README には以下も書いてください:
- Windows Docker Desktop の Settings は画面右上の歯車アイコンから開く
- General で “Use the WSL 2 based engine” を確認する
- Resources > WSL integration で “Enable integration with my default WSL distro” と実際に使う Ubuntu distro のトグルを有効化する
- `docker` コマンドが WSL2 で見えない場合の対処
- 3000/3001 が既に使われている場合は `GRAFANA_PORT=3030` のように変える
- Open WebUI のログイン画面が出る場合は、そのポートは Grafana ではないと説明する

最後に、JSON構文、Bash構文、docker compose config を検証してください。

短縮版プロンプト

短く依頼したい場合は、こちらでも十分です。

Grafana REST API を使って、WSL2 + Docker Desktop 上に TestData datasource と IoTセンサー監視風ダッシュボードを自動作成する実装を作ってください。Docker Compose、セットアップ用 Bash スクリプト、dashboard JSON、README を作成し、ポート競合に備えて GRAFANA_PORT で公開ポートを変更できるようにしてください。ダッシュボードには温度、燃圧、RPMゲージ、気圧Stat、アラート履歴Tableを配置してください。Windows Docker Desktop の WSL Integration 手順もREADMEに含めてください。
必要に応じて、ポート番号、ダッシュボード名、パネル名、datasource種別を変更してください。
