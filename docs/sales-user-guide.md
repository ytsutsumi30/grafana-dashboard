# 営業担当者向け Grafana Cloud ダッシュボード提案ツール 利用ガイド

## 1. このツールでできること

このツールは、訪問先の製造業顧客に合わせてGrafana Cloudのデモダッシュボードを作成するための営業支援ツールです。

たとえば、以下のような入力からダッシュボード案を作成できます。

- 板金加工業者
- プレス加工業者
- 射出成形業者
- 食品加工業者
- 半導体関連製造業
- 表面処理業
- 自動車部品製造業
- 化学製造業
- 医薬品製造業
- 物流倉庫IoT
- 電力監視IoTデバイス

作成されるダッシュボードは、実データではなくGrafanaのTestData datasourceを使ったデモ用データです。顧客との会話で「このような監視画面を作れます」と見せるために使います。

## 2. アクセスURL

PoC環境:

```text
https://grafana-dashboard-builder-pjvjufzh3q-an.a.run.app
```

作成先Grafana Cloud:

```text
https://ytsutsumi30.grafana.net
```

## 3. 基本的な使い方

### 3.1 業種を入力する

左側の入力欄に、訪問先顧客の業種や監視対象を入力します。

例:

```text
板金加工業者
```

または:

```text
電力監視IoTデバイス
```

### 3.2 ダッシュボード種別を選ぶ

次の2種類から選択します。

| 種別 | 用途 |
| --- | --- |
| 製造ライン・設備保全 | 機械、ライン、保全、異常兆候監視のデモ |
| IoTデバイス監視 | センサー、電力、通信状態、デバイス状態監視のデモ |

製造設備の提案では `製造ライン・設備保全` を選びます。

電力監視やセンサーデバイス提案では `IoTデバイス監視` を選びます。

`顧客/案件ラベル` と `営業担当者` は任意です。提案メモ印刷とブラウザ側の作成履歴に使われます。PoCでは、顧客の機密情報や個人情報を入力しないでください。

### 3.3 パネル案を作成する

必要に応じて `Dashboard folder` で作成先フォルダを選びます。未選択または `General / ルート` の場合はGrafanaのルートに作成されます。

Cloud Run側でアクセスコードが設定されている場合は、左側の `アクセスコード` に担当者向けに共有されたコードを入力します。入力したコードはブラウザのセッション中だけ保持されます。

`パネル案作成` ボタンを押します。

入力した業種に応じて、ダッシュボードに配置するパネル案が表示されます。

パネル案の作成方式は3種類あります。

| 表示 | 意味 |
| --- | --- |
| 既知業種テンプレート | あらかじめ用意された安定テンプレート |
| AI生成 | 未知業種に対してVertex AI Geminiで作成 |
| 汎用テンプレート | AIが使えない場合の代替案 |

### 3.4 パネル案を編集する

表示されたパネルは、作成前に編集できます。

右側上部の工程表示は、パネル案作成後に `パネル編集` へ進みます。左側では顧客条件と作成ボタンを常に確認でき、作成履歴やAIデモなどの補助機能は見出しを押して開きます。

編集できる項目:

- パネル名
- 可視化方式
- 単位
- 最新値のみ表示するか
- 最小値
- 最大値
- Warning 閾値
- Critical 閾値
- 目的

パネルの追加・削除もできます。

入力内容に問題がある場合は、該当パネルにエラーが表示され、`Grafana Cloud に作成` ボタンは押せません。

主なチェック項目:

- パネル名が入力されていること
- 最大値が最小値より大きいこと
- Warning 閾値がCritical閾値より小さいこと
- Warning / Critical 閾値が最小値と最大値の範囲内であること

### 3.5 生成前プレビューを確認する

パネル案を作成すると、右側上部に `生成前プレビュー` が表示されます。

プレビューでは、Grafana Cloudに作成する前に以下を確認できます。

- ダッシュボード全体のパネル配置
- Grafana風の時系列グラフ、Stat、Gauge、ドーナツ、Tableの画面イメージ
- パネル数
- 作成先Dashboard folder
- パネルごとの可視化方式
- Warning / Critical 閾値

パネル名、可視化方式、閾値、パネル追加・削除を変更すると、プレビューも更新されます。

スマートフォン表示ではプレビュー部分を横へ動かして、Grafanaの24カラム相当の配置を確認できます。ページ全体は横へはみ出しません。

編集内容は同じブラウザへ自動保存されます。誤って画面を更新した場合も、前回のパネル案、閾値、案件ラベルなどが復元されます。アクセスコードは下書きへ保存されません。最初から作り直す場合は右上の `下書き破棄` を押します。

### 3.6 提案メモを印刷する

パネル案を作成した後、右上の `提案メモ印刷` を押すと、提案メモ用の画面が開きます。

ブラウザの印刷画面で `PDFに保存` を選ぶと、顧客説明用の簡易提案メモとして保存できます。

### 3.7 Grafana Cloudに作成する

内容を確認したら、`Grafana Cloud に作成` ボタンを押します。

作成に成功すると、右側上部の工程表示が `Grafana作成` へ進み、作成先URLがリンクとして表示されます。

作成成功後、保存されていた編集中の下書きは削除されます。

### 3.8 AI故障診断デモを確認する

Androidスマホ振動センサーデモを使う場合、左側の `AI故障診断デモ` から直近の振動データを解析できます。

見せ方:

- Androidアプリでセンサー送信を開始する
- スマホを静止させた状態で `AI診断を実行` を押す
- スマホを振る、または画面をタップする
- 再度 `AI診断を実行` を押し、リスク、原因候補、推奨対応が変わることを見せる

この診断はデモ支援用です。実際の故障を確定するものではなく、振動傾向から保全担当者向けの確認ポイントを提示する機能です。

### 3.7.1 Android実機がない場合

Androidスマホを用意できない場合は、左側の `デモ波形生成` を使います。

使い方:

- `正常`、`注意`、`異常` のいずれかを選ぶ
- 必要に応じて点数を変更する
- 通常は `シナリオ実行` を押す

`シナリオ実行` は、過去データのリセット、波形生成、AI故障診断をまとめて行います。

個別に操作したい場合:

- デモ開始前に `波形をリセット` を押す
- `波形を生成` を押す
- GrafanaのAndroidデモダッシュボードを更新する
- `AI Maintenance Insight` と `AI App Log Analysis` の変化を確認する

営業デモでは、まず `正常` を生成して安定状態を見せ、その後 `注意` または `異常` を生成すると、波形とAIコメントの変化を説明しやすくなります。

### 3.9 AIログ解析デモを確認する

左側の `AIログ解析デモ` では、営業デモ中のアプリ操作ログをAIで要約できます。

確認できること:

- パネル案作成やダッシュボード作成が正常に動いたか
- APIエラーが発生していないか
- エラーがある場合の原因候補
- 次に確認すべき設定や操作

これはCloud Runの公式ログを直接読む機能ではありません。PoCではアプリ内に保持している直近イベントを解析します。

作成が完了すると、画面左側に以下が表示されます。

```text
名前：sheet-metal-maintenance-demo

Grafana Cloud に板金加工業者保全用ダッシュボードを作成しました。
URL:
https://ytsutsumi30.grafana.net/d/sheet-metal-maintenance-demo/sheet-metal-machine-maintenance-demo
```

表示されたURLをクリックすると、作成されたGrafana Cloudダッシュボードへ移動できます。

左側の `作成履歴` には、この画面から作成した直近のダッシュボードURLが表示されます。画面を更新した後でも、Cloud Runインスタンスが同じ間は `履歴を更新` でURLを再表示できます。

同じPC・同じブラウザでは、直近の作成履歴がブラウザにも保存されます。Cloud Runの再起動後でも、ブラウザ側に残っている履歴は表示できます。

## 4. 上書きチェックボックスの使い方

`既存ダッシュボードを上書きする` チェックボックスで、既存ダッシュボードの扱いを選べます。

### 4.1 チェックしない場合

同じ名前のダッシュボードがすでにある場合、新しい名前で作成されます。

例:

```text
sheet-metal-maintenance-demo
sheet-metal-maintenance-demo_1
sheet-metal-maintenance-demo_2
```

営業デモでは、通常はこちらを使います。過去に作成したデモを消さずに残せます。

### 4.2 チェックする場合

同じ名前の既存ダッシュボードを上書きします。

同じ案件で内容を調整し直す場合に使います。

誤って既存デモを更新したくない場合は、チェックを外してください。

## 6. 可視化方式の選び方

| 可視化方式 | 向いているデータ |
| --- | --- |
| 時系列 | 温度、電流、振動、圧力、サイクルタイムなどの推移 |
| Stat | 現在温度、現在湿度、現在消費電力など最新値 |
| Gauge | 圧力、発電量、不良率など範囲内の現在値 |
| ドーナツ | 設備別比率、カテゴリ別構成比 |
| Table | デバイス通信状態、アラート一覧、設備一覧 |

迷った場合は、時間変化を見る項目は `時系列`、現在値だけでよい項目は `Stat` を選びます。

製造ライン・設備保全では、顧客に全体像を説明しやすくするため、業種別センサーの前に以下の共通KPIが追加されます。

| 共通KPI | 説明 |
| --- | --- |
| Overall Equipment Effectiveness | 設備総合効率の概況 |
| Availability / Uptime | ラインの現在稼働率 |
| Unplanned Downtime | 直近の計画外停止時間 |
| Active Alarm Count | 未解決アラーム件数 |
| Maintenance Action Queue | 保全担当が次に確認する設備、優先度、推奨対応 |
| Production Loss Breakdown | 停止、段取り、速度低下、品質ロスの構成比 |
| Shift Production Summary | シフト別の計画数、実績数、不良数、停止時間、達成率 |
| Quality Defect Trend | 時間帯ごとの不良率推移 |
| Top Defect Reasons | 主要な不良理由の件数構成 |
| MTBF / MTTR Trend | 平均故障間隔と平均復旧時間の推移 |
| Alert Rule Candidates | 本番化時に通知化する候補条件、重要度、通知先、一次対応 |

## 7. 閾値編集の考え方

各パネルでは `Warning 閾値` と `Critical 閾値` を編集できます。

| 閾値 | 意味 |
| --- | --- |
| Warning | 注意して確認したい値。Grafanaでは黄色系で表示されます |
| Critical | すぐ確認すべき値。Grafanaでは赤系で表示されます |

例:

```text
Cycle Time: 10-15秒
Warning: 14秒
Critical: 15秒
```

顧客訪問時は、実際に現場で「注意」と「異常」にしている値を聞き、作成前に編集すると提案画面が具体的になります。

## 8. 顧客訪問時の使い方

### 8.1 商談前

1. 顧客の業種を調べる
2. 近い業種名でパネル案を作成する
3. 顧客に関係が薄いパネルを削除する
4. 顧客が関心を持ちそうな指標を追加する
5. Grafana Cloudに作成してURLを控える

### 8.2 商談中

1. ダッシュボードURLを開く
2. 各パネルが何を監視するか説明する
3. 顧客の実際の設備・センサー名を聞く
4. その場でパネル名や目的を修正する
5. 実データ化する場合に必要なデータソースを確認する

### 8.3 商談後

1. 作成したURLを案件メモに残す
2. 顧客から聞いた設備名・センサー名を整理する
3. 本番データソース候補を確認する
4. 必要であれば上書きチェックを使ってダッシュボードを更新する

## 9. 顧客ヒアリング項目

ダッシュボード案を作った後、以下を確認すると次の提案につなげやすくなります。

| 確認項目 | 質問例 |
| --- | --- |
| 監視対象 | どの設備やラインを優先して見たいですか |
| 取得データ | 温度、電流、振動、圧力など、現在取れているデータはありますか |
| 更新頻度 | 何秒または何分間隔で見たいですか |
| 異常判定 | どの値を超えると注意または異常ですか |
| 利用者 | 保全担当、製造管理者、工場長の誰が見ますか |
| 通知 | 異常時にメールやチャット通知が必要ですか |
| 既存システム | PLC、SCADA、MES、Excel、DBなど既存のデータ元はありますか |

## 10. 注意事項

- このツールで作成するデータはデモ用のモックデータです
- 実際の顧客設備データとは値が異なります
- AI生成されたパネル案は、顧客の業務に合わせて確認・編集してください
- 一般公開中のPoC URLは、URLを知っている人が利用できる状態です
- 不要なダッシュボードを大量に作成しないよう注意してください
- 既存ダッシュボードを残したい場合は、上書きチェックを外してください

## 11. よくある質問

### Q. パネル案作成で「汎用テンプレート」と表示されました

Vertex AI GeminiまたはAI生成に失敗した場合に表示されます。デモ作成は可能ですが、業種への最適化は弱くなります。

### Q. 同じ名前のダッシュボードが増えました

上書きチェックを外している場合、既存ダッシュボードを守るために `_1`, `_2` 付きで新規作成されます。

### Q. 作成されたURLをクリックしてもGrafanaに入れません

Grafana Cloud側のログインが必要です。Grafana Cloudにアクセスできるアカウントでログインしてください。

### Q. 顧客名を入れてもよいですか

PoCでは業種名や監視対象名の入力を推奨します。顧客の機密情報や個人情報は入力しないでください。

### Q. 実データに切り替えられますか

可能です。ただし、現在のツールはTestData datasourceを使ったデモ作成用です。実データ化する場合は、Grafanaのデータソース設定と各パネルのクエリ変更が必要です。

左側の `実データソース差し替え` で `データソース取得` を押すと、Grafana Cloudに登録済みのデータソースを選べます。`差し替え表を作成` を押すと、パネルごとに対象データソース、クエリ例、確認点が表示されます。これは実装前の確認表であり、実データのクエリ自動変換ではありません。

## 12. トラブル時の確認

### 画面が開かない

Cloud RunのURLが正しいか確認してください。

```text
https://grafana-dashboard-builder-pjvjufzh3q-an.a.run.app
```

起動状態だけ確認する場合は、以下を開きます。

```text
https://grafana-dashboard-builder-pjvjufzh3q-an.a.run.app/api/ping
```

画面左側の `運用診断` で `設定状態を確認` を押すと、Grafana token設定有無、AI provider、レート制限、アプリ内イベント数を確認できます。

### ダッシュボード作成に失敗する

以下の可能性があります。

- Grafana Cloud Tokenの期限切れまたは権限不足
- Grafana Cloud側の一時的なエラー
- TestData datasource作成権限がない
- ネットワーク接続の問題

### パネル案作成に失敗する

以下の可能性があります。

- Vertex AI APIの無効化または権限不足
- Vertex AI Geminiの一時的な利用制限
- 入力内容が空欄

既知業種テンプレートが使える入力で再試行してください。

## 13. 推奨する営業デモの流れ

1. 顧客業種を入力する
2. ダッシュボード種別を選ぶ
3. パネル案を作成する
4. 顧客に不要な項目を削除する
5. 顧客に響く項目を追加する
6. Grafana Cloudに作成する
7. URLを開いて画面を見せる
8. 実データ連携に必要なセンサー・データソースをヒアリングする

この流れにより、抽象的なIoT提案ではなく、顧客の業務に近い監視画面を見せながら会話できます。
